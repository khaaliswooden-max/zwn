# Contributing to ZWN

This is a Zuup Innovation Lab (ZIL) internal repository. External pull requests are
not accepted without prior discussion — reach out to khaaliswoeden@gmail.com first.

---

## Branching Convention

| Prefix | When to use |
|---|---|
| `claude/<scope>-<slug>` | Claude Code sessions (auto-generated by the harness) |
| `feat/<description>` | Manual feature work |
| `fix/<description>` | Manual bug fixes |
| `chore/<description>` | Maintenance, dependency updates, config |
| `docs/<description>` | Documentation-only changes |

All branches target `main` via pull request. Direct pushes to `main` are disabled.

---

## Commit Style

Follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/):

```
feat(indexer): add Symbion listener and BiologicalReading parser
fix(frontend): prevent render loop in NebulaCanvas on hot reload
chore(deps): upgrade @coral-xyz/anchor to 0.30.1
docs(CLAUDE.md): add QAL session prompt
refactor(causal): extract rule evaluator into pure function
```

Types in use: `feat`, `fix`, `chore`, `docs`, `refactor`

## Prerequisites

| Tool | Version | Install |
|---|---|---|
| Node.js | 18+ | https://nodejs.org/ |
| Rust | stable | https://rustup.rs/ |
| Anchor CLI | 0.30.1 | `cargo install --git https://github.com/coral-xyz/anchor avm && avm install 0.30.1` |
| Solana CLI | latest | https://docs.solana.com/cli/install-solana-cli-tools |
| Neo4j | 5+ | https://neo4j.com/download/ or Docker (see below) |

**Neo4j via Docker (quickest local setup):**

```bash
docker run \
  --name neo4j-zwm \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/your_password_here \
  neo4j:5
```

```bash

cp zuup-zwm-indexer/.env.example zuup-zwm-indexer/.env
# Edit zuup-zwm-indexer/.env — fill in NEO4J_PASSWORD, any missing program IDs

# Daily brief
cp zwm-daily/.env.example zwm-daily/.env
# Edit zwm-daily/.env — add ANTHROPIC_API_KEY
```

Never commit `.env` files. All are covered by `.gitignore`.

## Building Each Package

**Root (builds frontend for Vercel):**

```bash
npm run build
```

**Indexer:**

```bash
cd zuup-zwm-indexer
npm install
npm run build
npx ts-node src/db/init.ts   # run once before first startup
npm start
```

**Frontend:**

```bash
cd frontend
npm install
npm run dev        # development: http://localhost:3000
npm run build      # production build
```

**SDK (`@zuup/zwm-sdk`):**

```bash
cd zuup-zwm-indexer/sdk
npm install
npm run build      # outputs to sdk/dist/
```

**Anchor (per platform repo — run from that repo's root):**

```bash
# Example: civium
cd ~/your-workspace/civium
anchor build
# Then copy the new IDL:
cp target/idl/civium.json ~/your-workspace/zwn/zuup-zwm-indexer/idl/
```

## Claude Code Session Workflow

`CLAUDE.md` defines 9 backend sessions and 4 frontend sessions, each with a single scope.
Run `claude` from the workspace root (`~/your-workspace/`), not from inside any sub-repo.

Sessions are designed to be run in order (Session 1 → 9, then UI-1 → UI-4). Each session
produces a PR; merge before starting the next session to keep context clean.

Refer to the **Session Prompts** section in `CLAUDE.md` for the exact prompt to use for
each session.

## Testing

**Indexer unit + integration tests:**

```bash
cd zuup-zwm-indexer
npm test
```

**Green-path e2e (Civium → Aureon causal chain, requires devnet + Neo4j):**

```bash
cd zuup-zwm-indexer
npx ts-node tests/civium-e2e.ts
```

The e2e test validates the full pipe: Solana event emission → listener → parser →
Neo4j write → causal engine → Aureon ingest → Neo4j causal chain query. All 7 steps
must pass before adding additional platform listeners.
